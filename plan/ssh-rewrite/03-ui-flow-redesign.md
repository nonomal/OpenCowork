# SSH 页面流程重新设计

> 前置阅读：`01-current-design-analysis.md`（第 5 节）、`02-rewrite-design.md`（第 5 节）。
> 本文档回答"用户怎么走"：新建、连接、断线、多终端、SFTP、导入的完整交互流程。

## 1. 现状流程的问题

现状 SshPage 是**三个全屏互斥模式**的状态机（library → connect stage → connected workspace）：

1. **点击连接后整页切走**：连接进行中和连接成功后都看不到主机库；想连第二台主机必须先"返回"离开当前终端视图。
2. **一个终端 tab = 一条完整 TCP 连接**：同一主机开第二个终端会重新拨号、重新认证一遍。
3. **连接失败/掉线是终点**：失败只有全屏错误页；掉线终端直接死掉，scrollback 丢失，回库页从头再来。
4. **SFTP 与终端割裂**：SFTP 双窗格挂在"支持工作区"里，与连接工作区平行，同一台主机的终端和文件在两个不同导航层级里。
5. **新建/编辑是 441 行平铺表单**：无内联连接测试反馈，错误信息与字段脱节。

## 2. 新信息架构：左 Rail + Tab 工作区

连接不再全屏接管页面。**一切皆 Tab**，connecting/失败/掉线都是 tab 内的状态，而不是页面模式。

```
┌──────┬─────────────────────────────────────────────┬─────────┐
│ Rail │ Tab 栏: [主机库] [web-1 ⚡] [web-1:SFTP] [db-2 ⟳]│ Inspector│
│      ├─────────────────────────────────────────────┤ (按需)   │
│ 主机 │                                             │          │
│ 密钥 │          活跃 Tab 内容                        │ 新建/编辑 │
│ 转发 │   (主机库 / 终端 / SFTP / 编辑器 / 监控)        │ 连接详情  │
│ 已知 │                                             │          │
│ 日志 │                                             │          │
│      ├─────────────────────────────────────────────┤          │
│      │ 底部传输中心（有活跃任务时展开）：进度/取消/重试   │          │
└──────┴─────────────────────────────────────────────┴─────────┘
```

- **Rail**：主机库（默认 tab，不可关）、密钥链、端口转发、已知主机、日志。点击 = 激活/新开对应 tab。
- **Tab 类型**：`library | terminal | sftp | sftp-compare | editor | monitor`。tab 标题带连接状态图标（⚡connected / ⟳reconnecting / ✕failed / spinner connecting）。
- **Inspector**：右侧抽屉，承载新建/编辑/详情，不遮挡工作区。
- **传输中心**：全局底部条，聚合所有上传/下载/远程复制任务（替代散落的任务列表）。
- 保留：SshWindowApp 独立窗口入口、主题 tone 系统（tone 跟随活跃 tab 类型而非页面模式）。

## 3. 新建/保存流程

```
入口A: 主机库 [＋新建]            入口B: 快速连接栏 "user@host[:port]"
   │                                │ 回车
   ▼                                ▼
右侧 Inspector 打开（三段式）      直接开 terminal tab 连接（临时连接）
   │                                │ 成功后 toast: [保存为主机?]
   ▼                                ▼ 点击 → 进入入口A 表单（已预填）
┌─ 基本 ────────────────────────┐
│ 名称 / 主机 / 端口 / 用户名 / 分组 │
├─ 认证 ────────────────────────┤
│ ○密码 ○私钥 ○Agent            │
│ [密码输入]  已保存时显示 "●●● 已设置 [更换][清除]"（只写不读）
│ 私钥: 路径选择 + 检测 ~/.ssh 默认密钥 + [安装公钥到远端]
│ [测试连接] ← 内联执行，分层结果就地显示：
│   ✓ 可达 → ✓ 认证通过   或   ✗ "跳板机认证失败: ..."（错误贴在对应段）
├─ 高级（默认折叠）──────────────┤
│ proxyJump（下拉可选已存主机 / 手输 alias / user@host:port）
│ 启动命令 / 默认目录 / keepalive
└───────────────────────────────┘
   [保存]   [保存并连接]（主按钮）
```

保存后：库列表增量更新（delta 事件，非全量 reload），新卡片高亮 2s。

## 4. 连接流程（用户视角状态机）

```
库页点 [连接] ──► 新建 terminal tab（若该主机已有活跃 Handle：直接复用，秒开）
                    │
                    ▼
        tab 内三阶段指示（非全屏）：
        ①拨号(跳板逐级显示) → ②认证 → ③启动 shell
                    │
     ┌──────────────┼────────────────┐
     ▼              ▼                ▼
   ready          失败             用户中途取消
   终端可用      tab 内错误卡片:      tab 关闭
   badge ⚡      分层原因 + 建议
                [重试] [编辑连接] [查看日志]
```

连接后的生命周期：

```
ready ──掉线──► reconnecting（tab 内顶部横幅，指数退避自动重连）
   ▲              │  终端保留 scrollback，输入禁用
   │              │  横幅: "连接已断开，正在重连(第2次)… [立即重试][停止]"
   │  重连成功，注入提示行 "── 已重连 ──"，SFTP 懒重建
   │              ▼
   └───────── 重连放弃/手动断开 ──► disconnected
                                    终端内容保留 + 居中 [重新连接] 按钮
                                    tab 可继续翻阅 scrollback，关 tab 才释放
```

要点：
- **同主机多终端**：`web-1` 已连接时再点 [连接] 或 tab 栏 [＋]，在同一 ConnectionHandle 上开新 shell channel——无二次认证，tab 命名 `web-1 (2)`。
- **库页卡片显示活跃徽标**：`⚡2 终端 · SFTP`，并提供 [新终端] [文件] [断开全部] 快捷操作。
- 密码类连接若解密失败/未存密码：连接前弹一次性密码输入（会话内不落盘）。

## 5. SFTP / 文件流程

- **单窗格 SFTP tab**：终端 tab 工具栏 [文件] → 开当前连接的 `sftp` tab（复用同一 Handle，无需再连）。目录树 + 面包屑 + 拖拽上传；双击文本文件 → `editor` tab（保留 beforeContent 乐观锁）。
- **双窗格对比 tab**（`sftp-compare`）：库页 [传输] 入口，左右各选主机（或本地），选中即触发 Handle 复用/建连。拖拽/多选 → 上传/下载/远程复制。
- **冲突策略**：任务发起时就地小弹层选择（跳过/覆盖/重命名），不再是全局隐藏设置。
- **传输中心（底部条）**：所有任务聚合，行内 [取消]；失败任务保留 [重试]（同参数重发）；完成 30s 后自动淡出，可展开历史。

## 6. 导入 / 密钥链流程（微调保留）

- 导入：库页 [导入] → 选源（open-cowork JSON / OpenSSH config）→ 预览表格（冲突行内选 create/skip/replace/duplicate）→ 应用 → 回库页高亮新增行。语义不变，实现由 sidecar 迁到主进程 TS。
- 密钥链：Rail 入口，保留本地 ~/.ssh 扫描、生成、[安装公钥到主机]；安装动作复用共享 exec channel（不再 spawn 本地 ssh）。

## 7. 与里程碑的对应（渐进切换，避免 UI 大爆炸）

| 里程碑 | UI 变化 |
|---|---|
| M1 | 连接状态进 tab（connect stage 不再全屏）；重连横幅；同主机多终端复用 Handle |
| M2 | SFTP tab 化 + 冲突就地选择 + 底部传输中心 |
| M3 | 左 Rail + tab 工作区正式布局；Inspector 三段式表单 + 内联测试 + secret 占位；快速连接保存引导；库页活跃徽标 |
| M4 | 旧三模式代码 & 支持工作区容器删除 |
