# 项目介绍 / Introduction

OpenCowork 是一个开源的桌面 AI Agent 工作台。它不是单纯的聊天应用，而是把 **对话、计划、任务、子代理、团队协作、消息平台、MCP、Cron、SSH、文件预览** 放到同一个本地运行时里。

## 它解决什么问题 / What it solves

很多 AI 工具只会“回答问题”，但很难稳定地完成真正的工作。OpenCowork 的目标是把“思考”变成“执行”：

- 读写本地文件
- 运行 Shell / Git / 搜索命令
- 调用 MCP 工具
- 通过计划模式把复杂任务拆成可审阅的步骤
- 通过子代理并行处理不同子任务
- 通过 Cron 把一次性的能力变成持续的自动化
- 通过消息平台把结果推送到真实工作场景

## 核心设计 / Core design

| 设计点 | 含义 |
| --- | --- |
| Local-first | 数据、配置、运行时尽量都落在本机 |
| Multi-agent | 主 Agent 可以调度子代理和团队成员 |
| Tool-driven | 所有非平凡工作都通过明确的工具完成 |
| Evidence-based | 文档和运行时都尽量围绕真实源码和真实 IPC 契约 |
| Secure bridge | renderer 不直接碰系统能力，系统能力放在 main |

## 典型工作流 / Typical flows

### 1. 日常对话

使用 **Chat** 或 **Cowork** 模式进行常规问答、文档处理、信息检索。

### 2. 代码修改

切换到 **Code** 模式，设置 working folder，然后让 Agent 读取项目、搜索文件、修改代码、运行命令。

### 3. 不确定需求

切换到 **Clarify** 模式，让 Agent 先收集信息，再进入 **Plan Mode** 输出可审阅计划。

### 4. 架构设计

切换到 **ACP** 模式，让主 Agent 只负责设计、拆解和分派，不直接修改代码。

### 5. 自动化

用 **Cron**、**MCP**、**Plugins**、**Skills** 和 **Sub-Agents** 把重复工作自动化。

## 你需要记住的两件事

1. **OpenCowork 的强项是执行，不只是生成文本。**
2. **OpenCowork 的可靠性来自明确边界：UI、工具、主进程、共享协议各司其职。**
