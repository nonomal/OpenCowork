# 提供商配置概述 / Providers Overview

OpenCowork 的提供商体系分两层：

1. **运行时 adapter**：真正负责发请求、收流、处理 image / thinking / reasoning 的底层实现
2. **provider preset**：给用户配置页面和模型列表使用的预设项

## 当前实现 / Current implementation

| 层 | 数量 | 说明 |
| --- | --- | --- |
| 运行时 adapter | 5 | `anthropic`、`openai-chat`、`openai-responses`、`openai-images`、`gemini` |
| 内置 provider preset | 24 | 覆盖 Anthropic、OpenAI、Google、Ollama、DeepSeek、Qwen、Moonshot、MiniMax、Baidu、SiliconFlow 等 |

## 支持的主要提供商 / Main provider families

| 提供商 | 入口 | 说明 |
| --- | --- | --- |
| [Anthropic](/docs/providers/anthropic) | native | Claude、thinking、vision |
| [OpenAI / Azure OpenAI](/docs/providers/openai) | `openai-chat` / `openai-responses` | Chat、reasoning、image generation |
| [Google Gemini](/docs/providers/google) | `openai-chat` | 超长上下文、多模态 |
| [DeepSeek](/docs/providers/deepseek) | `openai-chat` | 性价比和推理模型 |
| [Ollama](/docs/providers/ollama) | `openai-chat` | 本地运行、无需云 API Key |
| [其他提供商](/docs/providers/other-providers) | mostly `openai-chat` | Qwen、Moonshot、MiniMax、Baidu、SiliconFlow、OAuth preset 等 |

## 统一配置 / Unified config

每个提供商页都可以理解成同一套字段：

- provider type
- Base URL
- API Key / OAuth
- model list
- category（chat / image）
- vision / thinking / reasoning flags

## 什么时候选哪类？ / Which one should I choose?

- 想要原生思考和最稳的 Claude 体验：选 Anthropic
- 想要最通用的 API 兼容：选 OpenAI-compatible preset
- 想要本地离线：选 Ollama
- 想要超长上下文：选 Gemini
- 想要便宜且适合中文：看 DeepSeek / Qwen / SiliconFlow / Baidu 等 preset

## 图像模型 / Image models

图像模型通常会被标成 `category = image`。这样图像插件才会把它识别成可用的生成源。

## 配置入口 / Where to configure

在应用里：

- `Settings → AI Provider`
- `Settings → Model management`
- `Settings → Model`

不同 preset 的文件都放在：

```text
src/renderer/src/stores/providers/
```
